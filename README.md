# 엄마를 위한 로또 번호 추첨기 ✨

> 💝 실제 1~1222회 동행복권 당첨 데이터 기반 분석 + 가중치 알고리즘

React + Vite로 만든 데이터 기반 로또 번호 생성 웹 애플리케이션입니다.

---

## 🎯 주요 기능

- **실제 데이터 기반 분석** — 동행복권 1회(2002-12-07) ~ 1222회(2026-05-02) 당첨 번호 분석
- **4가지 추첨 방식** — 핫 추천 / 콜드 역발상 / 균형 조합 / 완전 랜덤
- **실시간 통계** — 합계, 평균, 홀짝, 연속 번호
- **데이터 인사이트 패널** — Top/Bottom, 최근 핫넘버, 분포 시각화
- **반응형 UI** — 모바일/태블릿/데스크톱 지원

- **4중 교차검증**: 4개 독립 미러로 1~1222회 모두 대조 확인 — `hjleesm/lottery-data`, [`happylie/lotto_data`](https://github.com/happylie/lotto_data), [`z1533226z-max/lotto-website`](https://github.com/z1533226z-max/lotto-website), [`jdyang88/Korea_Lotto`](https://github.com/jdyang88/Korea_Lotto)
- **발견된 오류**: `happylie/lotto_data`에 1070/1071회가 서로 뒤바뀌어 있음 (다른 3개 미러는 일치). 본 빌드는 정답인 `hjleesm`을 채택하므로 별도 보정 코드 불필요.

### 데이터 범위
**1회 (2002-12-07) ~ 1222회 (2026-05-02), 총 1,222회차**

### ⚠️ 데이터 한계 (정직한 공개)
- 본 빌드는 1222회까지 사용했습니다.
- 1223회(2026-05-09), 1224회(2026-05-16)는 미포함입니다 — 동행복권 공식 사이트(`www.dhlottery.co.kr`)가 작업 시점 기준 점검 중("사이트 준비중입니다")으로 어떤 미러도 신규 회차를 수집하지 못하는 상태였습니다.
- 사이트 복구 시 `npm run data:update` 한 줄로 최신화됩니다.

### 데이터 갱신 방법
```bash
npm run data:fetch    # 미러 + 공식 API에서 원시 데이터 갱신
npm run data:analyze  # 통계 재계산
npm run data:update   # 위 두 단계 한 번에
```

## 🎲 추첨 방식 4종

각 방식의 **실제 당첨 확률은 모두 동일합니다 (1/8,145,060)**.
다만 어떤 "이야기"로 번호를 고르고 싶은지에 따라 선택하세요.

### 🔥 핫 추천 (Hot)
- **로직**: 1222회 전체 출현 빈도 × 1 + 최근 50회 출현 빈도 × 8
- **선호 번호**: 34(182회), 27(180회), 12(177회), 13(176회), 45(174회) 같은 자주 나온 번호
- **추가 부스트**: 최근 50회 트렌드 번호에 강한 가중치

### ❄️ 콜드 역발상 (Cold)
- **로직**: `(최대빈도 - 빈도 + 30)` 역가중치
- **선호 번호**: 9(133회), 32(143회), 22(143회) 같이 오래 안 나온 번호
- **근거**: "확률은 같은데, 너무 안 나왔으니 곧 나올 차례" 라는 발상 (도박사의 오류이지만 인간적 선택)

### ⚖️ 균형 조합 (Balanced)
실제 1222회 분포에 통계적으로 부합하는 번호 조합을 생성합니다.
조건을 모두 만족할 때까지 최대 200회 재추첨:

| 조건 | 기준 | 실제 분포 |
|---|---|---|
| 활용 구간 수 | 4개 이상 (1-9, 10-18, 19-27, 28-36, 37-45 중) | 4개=55.5%, 5개=12.9% |
| 홀짝 비율 | 2:4 ~ 4:2 | 82.1% |
| 합계 | 105 ~ 175 | 약 75% |
| 연속쌍 수 | 0 ~ 2 | 98.3% |

### 🎲 완전 랜덤 (Random)
- `Math.random()` 순수 무작위
- 가장 "공정한" 방법 (확률론적으로 가장 정확)

### 출현 빈도
| 순위 | Top 5 | Bottom 5 |
|---|---|---|
| 1 | **34번** (182회) | **9번** (133회) |
| 2 | **27번** (180회) | 32번 (143회) |
| 3 | **12번** (177회) | 22번 (143회) |
| 4 | **13번** (176회) | 23번 (147회) |
| 5 | **45번** (174회) | 41번 (148회) |

→ 49회 차이는 통계적으로 유의미하지만, 실제 확률에는 영향 없음 (총 1222회의 무작위 변동 범위)

### 합계
- 평균: **138.23** (이론값 138과 거의 일치)
- 표준편차: 30.78
- 중앙 50% 구간: 117 ~ 159

### 홀짝 비율 (가장 흔한 5개)
- **3:3 → 33.6%**
- 4:2 → 26.7%
- 2:4 → 22.2%
- 5:1 → 8.0%
- 1:5 → 6.7%

### 연속 번호
- 0쌍 → 48.4% (가장 흔함)
- 1쌍 → 39.1%
- 2쌍 → 10.8%
- 3쌍 이상 → 1.7%

## 🚀 시작하기

```bash
# 의존성 설치
npm install

# (선택) 최신 데이터 갱신
npm run data:update

# 개발 서버
npm run dev

# 프로덕션 빌드
npm run build

# 빌드 미리보기
npm run preview
```

개발 서버는 기본적으로 `http://localhost:5173`에서 실행됩니다.
